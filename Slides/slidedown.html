SlideDown — Portable Slide Language (PSL) reference

Coordinates may be fractions of the slide (0–1) or pixels in a 1280×720 space. Values ≤ 1 are treated as fractions.

Structure

---Start a new slide.
!---Start a pocket slide: rendered in the editor and reachable by direct navigation (overview, thumbnails, ?slide=N), but skipped when advancing with space/n. It still executes, so !set and \${…} math there affect later slides. Built-in slide_hidden is 1 on these slides.
!if expr … !else … !endifConditional lines (nestable). Truthy = non-zero number or non-empty string; combine with comparisons, e.g. !if slide_num > 3 or !if \${x*2} == 10.
!import(theme.psl)Splice a PSL template file in at this point — perfect for shared !set/!all house styles. Fetched relative to the page (CORS applies); imports may nest but cannot add slides.
[shown text](https://…)Inline link, opens in a new tab.
!foreach i 1 6 [step] … !endforRepeat the body once per value (inclusive; step may be fractional or negative; nestable). \${i} works everywhere in the body — labels, coordinates, durations — so one loop can build and animate a whole family of elements. Use \${…} for computed bounds.
# Title / ## SubtitleBig centered title and subtitle (title slides).
### Slide titleHeading at the top of a regular slide.
- item /   - sub-itemBullets; indent 2 spaces per level. **bold**, *italic*, `code` work in any text.
% a commentIgnored.
...Pause point — wait for spacebar before revealing what follows.
>>>Chain — run what follows automatically when the previous animations finish.
```cpp … ```Fenced code block with syntax highlighting (cpp, python, javascript, bash, and other common languages; omit the language for plain code). A --- inside a fence does not start a new slide. Size with {w=…, size=…}.
| a | b |
|:--|--:|
| 1 | 2 |		Markdown table. The |---| row makes the first row a header; :--/--:/:-: set column alignment. Cells take inline markdown and \${…}.

Variables, expressions & deck styling

!set test 5Define any variable. Numeric values become numbers; anything else is a string. Reserved style names configure the deck: bg dim tint color accent font headfont (e.g. !set font Georgia, serif).
The value is \${test}\${…} anywhere in slide text substitutes an expression: variables, numbers, strings, + - * / % ( ). + concatenates strings.
\${test*2 + 1}Math works, with normal precedence.
\${test = test + 1}Assignment: prints the new value and updates the variable. Expressions run at parse time, top to bottom, so results are deterministic.
\${slide_num} · \${num_slides} · \${slide_hidden}Built-ins: current slide number, total slide count (pocket slides included), and whether this slide is hidden.
&nbsp; &mdash; &rarr; &#x1F600; · \${nbsp} · \${chr(8594)}Special characters: HTML entities work in any text (never tags — markup stays escaped), emojis paste straight in, \${nbsp} is a non-breaking space, and chr(n) gives any code point.
\${min(3,7)} · \${sin(pi/2)} · \${test > 4}Functions: min max sin cos tan exp log log2 sqrt abs floor ceil round (with built-in pi). Comparisons < <= > >= == != return 1 or 0.
\${pic.x} · \${box.cy} · \${arr.x2}Members of any [label]ed element (values as declared, in px): all elements have x y (+ w h cx cy when known); lines/arrows add x1 y1 x2 y2; circles r; ellipses rx ry. Great for attaching one element to another.
!all
  …lines…		Everything after !all (to the end of that slide) is pasted at the top of every slide, then evaluated per slide — so \${slide_num} is correct. Build footers, headers, watermarks, running counters. One block per deck.
!set coords absoluteUndistorted geometry: fractions on both axes map to the slide height (one unit = 720px), so !rect (…)(0.3,0.3) is a true square. The slide is aspect (≈1.778) units wide; pixels (values > 1) are always undistorted. !set coords fraction restores the default (x → width, y → height). Built-ins: slide_w slide_h aspect.

Placement, labels, styling (prefixes, any order, before content)

@(0.55, 0.1) text or elementAbsolute position: top-left anchor at 55% across, 10% down. Add {anchor=center} to center on the point.
[mylabel] elementName an element so animations can target it.
{size=44px, color=crimson} textStyle overrides. Keys: size color font fill w h thick align valign anchor opacity · shapes/boxes also: textcolor pad round tint dim.

Images, backgrounds & embeds

![caption](url)Image. Size with {w=0.3} (fraction of slide width) or pixels.
!bg(url) · !bg(#123456)Slide background image (covers slide) or color.
{dim=0.5, tint=#000} !bg(url) · !dim 0.5Darken (or tint) the background so text on top stays readable. dim is 0–1.
!embed(https://your.app)Embed a live web app as an iframe. Position with @(), size with {w=, h=}. Thumbnails show a placeholder.

Shapes & text boxes

!line (x1,y1) (x2,y2)Line. Style with {color=…, thick=…}.
!arrow (x1,y1) (x2,y2)Arrow (head at second point).
!circle (cx,cy) rCircle; r is a fraction of slide height or pixels.
!ellipse (cx,cy) (rx,ry)Ellipse.
!rect (x,y) (w,h)Rectangle, top-left at (x,y).
!textbox (x,y) (w,h) Your textA rectangle with wrapped text inside. Style: fill, textcolor, size, align, valign, pad, round, thick (border).
!rect (x,y) (w,h) Some textAny closed shape (rect, circle, ellipse, polygon, diamond, star) can carry inner text — just type it after the geometry.
!polygon (x,y) (x,y) (x,y) …Closed polygon, any number of vertices. !triangle = 3-point shorthand; !polyline is the open version.
!diamond (cx,cy) (w,h) · !star (cx,cy) n r [r_in]Centered diamond; n-pointed star with outer radius r (inner defaults to 0.45·r).
!curve (x1,y1) (cx,cy) (x2,y2)Quadratic Bézier — ideal for diagram connectors; the middle pair is the control point.
!anim poly point 4 (x,y) 1spoint now animates any vertex of any multi-point shape. Members: polygons expose x y w h cx cy n plus per-vertex x1 y1 x2 y2 ….

Animations (target a [label]; duration optional, default 0.7s)

!anim pic hide 0.5s · !anim pic showFade out / fade in.
!anim pic move (0.6, 0.2) 1sMove to a new position. Shapes move by their first-listed point; flow text is lifted out of flow.
!anim arrow1 point 2 (0.8, 0.5) 1sMove just point 1 or 2 of a line/arrow.
!anim pic size 0.45 1sAnimate an image to a new width.
!anim pic scale 1.6 0.5sScale any element about its center.
!anim word color #cc2244 0.5sAnimate text color / shape stroke.
!anim word textsize 54px 0.7sAnimate font size.
{ease=bounce} !anim pic move …Easing on any animation: linear easein easeout back bounce elastic (default is ease in-out).
!anim pic path (cx,cy) (x,y) 1sMove along a quadratic Bézier from the current position via a control point.
!anim pic enter drop
!anim pic exit slideleft 0.4s		Entrance / exit effects: fade slideleft slideright slideup slidedown grow drop spin (exit also shrink). For an entrance, declare the element and its enter in the same step — it appears with the effect. drop defaults to a bounce.

Consecutive !anim lines run simultaneously. Separate with ... to wait for a keypress, or >>> to auto-run in sequence. The Insert ▸ Animation… menu builds these lines for you.

Presenting

space / → / clickAdvance one step (past a … or on to the next slide). Pressing during an animation finishes it instantly.
←Back one step.
BackspaceBack one whole slide.
n / pNext / previous slide (skipping remaining steps).
vSlide overview sidebar; click a slide to jump — including pocket (!---) slides, shown dashed.
⧉ Presenter viewOpens the slides in a separate window — drag it to the projector and press F (or click once) there for fullscreen. This window becomes your console: the script is your notes, auto-scrolled to the current slide, with thumbnails as a click-to-jump overview. Both windows' keyboards drive the show.
b · f / ⛶ · home / end · escBlackout · fullscreen · first/last slide · exit. The Present buttons enter fullscreen automatically; "▶ 2nd screen" (shown when an extended display is detected, Chromium browsers) presents fullscreen on your other monitor.

Printing & handouts

File ▸ Print / Save as PDF lays out one slide per page, fully advanced (all reveal steps shown), with pocket slides omitted and embeds shown as placeholders. The print stylesheet requests full-bleed 16:9 landscape pages. Enable “background graphics” if slide backgrounds look washed out; for physical paper, use “Fit to printable area” if your printer cannot print borderlessly.

Publishing a deck online

Host this HTML file and your .psl script anywhere (e.g. GitHub Pages), then share a link like slidedown.html?present=my_talk.psl — visitors land directly in the slideshow and can step through with space. Add &slide=5 to start on a specific slide. Cross-origin script URLs work if the host sends CORS headers; same-site files always work.

Editor ↔ preview. Moving the caret in the script selects the slide you're editing; clicking a thumbnail jumps the caret to that slide. Drag the divider between the panes to resize them; the ◂/▸ buttons on it collapse one side, and double-clicking it (or Ctrl/Cmd + \) toggles a full-width editor. The thin strip beside the thumbnails hides the rail. The layout is remembered between sessions.
