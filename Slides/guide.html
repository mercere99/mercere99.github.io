Building Slideshows with SlideDown

A practical guide to writing decks in the Portable Slide Language (PSL) — the plain-text format that drives slidedown.html. A deck is just a .psl text file (or text typed into the editor). This guide takes you from a one-line deck to animated diagrams, loops, and published links.

1. Getting started

There are three ways to run a deck:

  • Editor: open slidedown.html in a browser and type/paste PSL into the left pane. The preview updates live; moving the caret selects the slide you’re editing.
  • Load a file: use the file picker in the toolbar to open a .psl file, or click Sample to load the built-in tour.
  • Published link: host slidedown.html and your script together, then open slidedown.html?present=my_talk.psl to land straight in the slideshow (see §13).

Press ? in the app at any time for the built-in syntax reference. The logical slide is 1280 × 720; any coordinate ≤ 1 is a fraction of the slide, anything > 1 is pixels.

A minimal deck:

# My Talk
## An optional subtitle
---
### First real slide
- A point
- Another point

--- on its own line starts a new slide. Everything else is content.

2. Slide structure

Marker Meaning
--- Start a new slide (a line that is exactly three dashes).
!--- Start a pocket slide: shown in the editor/overview and reachable by direct navigation, but skipped when advancing with space/n. It still executes, so !set and ${…} there affect later slides.
# Title Big centered title (title-slide style).
## Subtitle Centered subtitle under a # title.
### Heading Heading at the top of a normal content slide.
- item Bullet. Indent 2 spaces per level for sub-bullets.
% comment Ignored.

A --- inside a fenced code block does not split the slide, so you can show code containing dashes safely.

3. Text content

Inline markup works in any text (titles, bullets, cells, shape labels):

  • **bold**, *italic*, `code`
  • HTML entities such as &mdash;, &rarr;, &nbsp;, &#x1F600; render as characters. Raw tags do not — markup always stays escaped, so you can’t inject HTML.
  • Emoji can be pasted directly. ${nbsp} is a non-breaking space; ${chr(8594)} yields any code point.

Code blocks are fenced and syntax-highlighted (cpp, python, javascript, bash, and other common languages; omit the language for plain code). Size them with a {w=…, size=…} prefix on the opening fence:

{w=0.56} ```cpp
#include <print>
int main() { std::println("hi"); }
```   ← closing fence

Tables use markdown pipes. A |---| separator row promotes the first row to a header; :-- / --: / :-: set column alignment. Cells accept inline markup and ${…}:

| Feature | Syntax  | Align |
|:--------|:-------:|------:|
| Bullet  | `- x`   | left  |
| Bold    | `**b**` | center|

4. Revealing content step by step

Two operators control pacing within a slide:

  • ... — a pause. Everything after it waits for the next spacebar/click.
  • >>> — a chain. What follows runs automatically when the previous animations finish.
### Build-up
- Shown immediately
...
- Revealed on the next keypress
  - This sub-bullet rides along (no ... of its own)
...
  - This one waits for its own keypress

Consecutive !anim lines run simultaneously; separate them with ... (wait for input) or >>> (auto-sequence) to stage them.

5. Placement, labels & styling (the three prefixes)

Any line can start with up to three prefixes, in any order, before its content:

Prefix Purpose
@(x, y) Absolute position. Top-left anchor at that point; add {anchor=center} to center on it.
[label] Name the element so animations and ${label.member} can target it.
{key=val, …} Style overrides. 
[pic] @(0.55, 0.18) {w=0.34} ![caption](image.png)

Style keys (all elements): size, color, font, fill, w, h, thick, align, valign, anchor, opacity. Shapes and boxes additionally take: textcolor, pad, round, tint, dim.

Without @(), text and bullets flow down the left of the slide as usual; positioned elements are lifted out of that flow.

6. The coordinate system

Values ≤ 1 are fractions; values > 1 are pixels in the 1280 × 720 space. Two modes control how fractions map:

  • !set coords fraction (default): x scales to slide width, y to slide height. Convenient for layout, but (0.3, 0.3) is a rectangle, not a square, because the axes have different spans.
  • !set coords absolute: fractions on both axes map to the slide height (one unit = 720 px), so !rect (…)(0.3,0.3) is a true square. The slide is aspect (≈ 1.778) units wide. Pixels (> 1) are always undistorted. Built-ins: slide_w, slide_h, aspect.

Switch modes mid-deck as needed and switch back with !set coords fraction.

7. Images, backgrounds & embeds

![caption](url)                         image; size with {w=0.3} (fraction) or pixels
!bg(url)   ·   !bg(#123456)             slide background image (covers) or color
{dim=0.55, tint=#0a1a2a} !bg(url)       scrim a busy background so text stays readable
!dim 0.4                                dim on its own line (0–1)
!embed(https://your.app)                embed a live web app as an iframe

!embed is positioned with @() and sized with {w=, h=} just like an image, and can be animated (move, scale, size, hide/show). Thumbnails show a lightweight placeholder; the live iframe appears in present mode.

8. Shapes & text boxes

All shapes take a {…} style prefix (color = stroke, fill, thick, etc.) and can be [label]ed and animated.

Shape Syntax
Line !line (x1,y1) (x2,y2)
Arrow !arrow (x1,y1) (x2,y2) — head at the second point
Circle !circle (cx,cy) r — r is a fraction of slide height or pixels
Ellipse !ellipse (cx,cy) (rx,ry)
Rectangle !rect (x,y) (w,h) — top-left at (x,y)
Text box !textbox (x,y) (w,h) Your text — a rect with wrapped text inside
Polygon !polygon (x,y)(x,y)(x,y)… — any number of vertices (closed)
Polyline !polyline (x,y)(x,y)… — the open version
Triangle !triangle (x,y)(x,y)(x,y) — 3-point shorthand
Diamond !diamond (cx,cy) (w,h) — centered
Star !star (cx,cy) n r [r_in] — n points, outer radius r (inner defaults to 0.45·r)
Curve !curve (x1,y1) (cx,cy) (x2,y2) — quadratic Bézier; middle pair is the control point

Any closed shape (rect, circle, ellipse, polygon, diamond, star) can carry inner text — just type it after the geometry, e.g. !circle (0.74,0.52) 0.2 centered label. Text boxes support fill, textcolor, size, align, valign, pad, round, and thick (border).

9. Animations

Target a [label]; the duration suffix (1.2s) is optional and defaults to 0.7 s.

Verb Effect
hide / show Fade out / fade in
move (x, y) Move to a new position (shapes move by their first-listed point)
point n (x, y) Move vertex n of any multi-point shape (line/arrow/polygon/…)
size w Animate an image/embed to a new width
scale f Scale any element about its center
color c Animate text color or shape stroke
textsize s Animate font size 
!anim dot move (0.75, 0.30) 1.2s
!anim arr point 2 (0.80, 0.55) 1.2s   ← these two run together (one keypress)
>>>
!anim dot color #c23b22 0.5s          ← starts when the move finishes

Timing model: consecutive !anim lines fire simultaneously. Put ... between groups to wait for input, or >>> to auto-chain. Pressing a key during an animation finishes it instantly.

10. Variables, expressions & functions

Define variables with !set; numeric values become numbers, anything else is a string:

!set test 5

${…} substitutes an expression anywhere in text — including labels, coordinates, and durations. Expressions evaluate at parse time, top to bottom, so results are deterministic.

  • Operators: + - * / % ( ) with normal precedence. + concatenates strings.
  • Assignment: ${test = test + 1} prints the new value and updates the variable.
  • Comparisons < <= > >= == != return 1 or 0.
  • Functions: min max sin cos tan exp log log2 sqrt abs floor ceil round, plus built-in pi.
  • Built-in variables: slide_num, num_slides (pocket slides included), slide_hidden, slide_w, slide_h, aspect.

Member access — any [label]ed element exposes its geometry (in px, as declared):

  • All elements: x y (plus w h cx cy when known)
  • Lines/arrows: x1 y1 x2 y2
  • Circles: r · Ellipses: rx ry
  • Polygons/polylines/curves: n plus per-vertex x1 y1 x2 y2 …

This is ideal for attaching one element to another:

[sq]  !rect (120, 260) (0.32, 0.32)
[rng] !circle (${sq.x + sq.w + 330}, ${sq.cy}) 0.16
      !arrow (${sq.x + sq.w + 14}, ${sq.cy}) (${rng.x - rng.r - 14}, ${sq.cy})

11. Loops & conditionals

Both are nestable and work over multiple lines. ${…} (including ${i}) is valid throughout the body, so a single loop can build and animate a whole family of elements.

!foreach — inclusive range; step may be fractional or negative:

!foreach i 0 5
[dot${i}] {fill=#1f6f6b} !circle (${180 + i*180}, ${560 - 80*sin(i*pi/5)}) ${14 + i*4}
!endfor
...
!foreach i 0 5
!anim dot${i} move (${180 + i*180}, 300) ${0.4 + i*0.15}s
!endfor

!if … !else … !endif — truthy means a non-zero number or non-empty string:

!if num_slides > 10
- Only shown when the deck has more than 10 slides
!else
- Shown otherwise
!endif

12. Deck-wide settings

Reserved !set names configure the whole deck’s default look:

!set bg #14212b
!set color #e8e4dc
!set accent #e2a84b
!set font Georgia, serif
!set headfont Helvetica, sans-serif
!set dim 0.3
!set tint #000

A slide’s own !bg overrides the deck default for that slide.

!all — everything after !all (to the end of that slide) is pasted at the top of every slide and re-evaluated per slide, so ${slide_num} is always correct. Use it for footers, page numbers, watermarks, and running counters. One !all block per deck:

!all
@(1225, 688) {size=18px, anchor=center} ${slide_num} / ${num_slides}
!if slide_hidden
@(640, 42) {color=#b03030, anchor=center} — POCKET SLIDE —
!endif

13. Presenting & publishing

Keyboard while presenting:

Key Action
space / → / click Advance one step (or finish a running animation)
← Back one step
Backspace Back one whole slide
n / p Next / previous slide (skip remaining steps)
v Slide overview; click to jump (pocket slides shown dashed)
b Blackout
f / ⛶ Fullscreen
home / end First / last slide
esc Exit

Presenter view: the ⧉ button opens the slides in a separate window — drag it to the projector and press F there for fullscreen. The original window becomes your console (auto-scrolled script as notes, thumbnails to click-jump). Both windows’ keyboards drive the show. On Chromium with an extended display, a “▶ 2nd screen” button presents fullscreen on the other monitor.

Publishing: host slidedown.html and your .psl file anywhere (e.g. GitHub Pages) and share:

slidedown.html?present=my_talk.psl        → opens straight into the show
slidedown.html?present=my_talk.psl&slide=5 → starts on slide 5

Same-site scripts always load; cross-origin URLs work if the host sends CORS headers.

14. Quick reference

STRUCTURE          --- new slide   !--- pocket slide   % comment
                   # Title   ## Subtitle   ### Heading   - bullet (2-sp indent)
                   ...  pause      >>>  auto-chain
PREFIXES           @(x,y)  [label]  {style}      (any order, before content)
STYLE KEYS         size color font fill w h thick align valign anchor opacity
                   + shapes/boxes: textcolor pad round tint dim
IMAGES/BG          ![cap](url)   !bg(url|#hex)   !dim 0.4   !embed(url)
SHAPES             line arrow circle ellipse rect textbox polygon polyline
                   triangle diamond star curve   (closed shapes hold inner text)
ANIMATE [label]    hide show   move (x,y)   point n (x,y)   size w
                   scale f   color c   textsize s     (dur suffix "1s", default 0.7s)
VARS/EXPR          !set name val    ${expr}    + - * / %    == != < <= > >=
                   min max sin cos tan exp log log2 sqrt abs floor ceil round, pi
                   built-ins: slide_num num_slides slide_hidden slide_w slide_h aspect
MEMBERS            label.x .y .w .h .cx .cy  ·  .x1..y2 (lines)  ·  .r  ·  .rx .ry  ·  .n
CONTROL            !foreach i a b [step] … !endfor    !if e … !else … !endif
DECK-WIDE          !set bg color accent font headfont dim tint
                   !set coords absolute | fraction     !all (per-slide overlay)
COORDS             ≤1 = fraction, >1 = pixels (1280×720)
PUBLISH            slidedown.html?present=deck.psl&slide=N

15. A complete small deck

# Digital Evolution
## a five-minute primer

!all
@(1225, 690) {size=16px, anchor=center} ${slide_num} / ${num_slides}
---
### What we'll cover
- Replication with variation
...
- Selection
...
- Open-ended novelty

---
!set coords absolute
### A simple feedback loop
[org]  {fill=#1f6f6b22, color=#1f6f6b, thick=4} !circle (300, 360) 0.16 organisms
[env]  {fill=#e2a84b22, color=#e2a84b, thick=4} !circle (960, 360) 0.16 environment
{thick=4} !arrow (${org.cx + org.r + 12}, ${org.cy - 40}) (${env.cx - env.r - 12}, ${env.cy - 40})
{thick=4} !arrow (${env.cx - env.r - 12}, ${env.cy + 40}) (${org.cx + org.r + 12}, ${org.cy + 40})
@(640, 250) {size=22px, anchor=center} act on
@(640, 470) {size=22px, anchor=center} selects
!set coords fraction
---
# Questions?
